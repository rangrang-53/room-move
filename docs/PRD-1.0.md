# 🏠 RoomMove PRD 1.0 (Prototype – Streamlit Ver.)

## 1. 프로젝트 개요
- **서비스명:** 룸무브(RoomMove)
- **목표:** 1인 가구가 이사 과정 전반(정보·일정·이삿짐 연동)을 간단히 관리할 수 있는 Streamlit 기반 웹앱
- **버전:** 1.0 (핵심 기능 검증용 프로토타입)
- **시작일자:** 2025-11-02
- **특징:** 별도 프론트/백 분리 없이, Streamlit으로 UI·로직·데이터를 한 환경에서 구현 및 배포

---

## 2. 문제 정의
1인 가구가 이사를 준비할 때 겪는 주요 문제:
- 해야 할 일이 많지만 **순서나 시점이 헷갈림** (전입신고, 계약 변경 등)
- **이삿짐 센터 비교나 예약 절차가 번거로움**
- 일정, 체크리스트, 문서가 흩어져 있어서 관리 피로도 높음

➡️ **룸무브는 “이사 To-Do를 자동 정리하고, 일정과 연결해주는 개인형 관리 웹앱”을 목표로 함.**

---

## 3. 타깃 사용자
| 구분 | 설명 |
|------|------|
| **1차 타깃** | 20~30대 1인 가구, 직장인, 자취생 |
| **특징** | 정보 탐색은 잘하지만 행정·일정 관리에 약함 |
| **목표 행동** | 룸무브 접속 → 이사 일정 등록 → 체크리스트 확인 → 이삿짐 정보 확인 |

---

## 4. 핵심 기능 (MVP 범위)
| 기능 | 설명 | 목표 |
|------|------|------|
| **① 이사 일정 등록 및 D-day 계산** | 사용자가 날짜를 입력하면 자동으로 D-day 계산 및 표시 | 이사 준비 흐름 시각화 |
| **② 체크리스트 자동 표시** | 전입신고, 공공요금, 청소 등 자동 리스트 제공 | 해야 할 일 명확화 |
| **③ 지역별 이삿짐 정보 표시** | 샘플 CSV 기반으로 지역별 이삿짐 업체 정보 노출 | 정보 탐색 효율화 |
| **④ 대화형 도움말(옵션)** | 간단한 질문에 Streamlit Chat 기능으로 응답 | 진입장벽 완화용 |

---

## 5. 사용자 여정 (핵심 시나리오)
1. 사용자가 웹페이지 접속  
2. `st.date_input()`으로 이사 날짜 입력  
3. 자동으로 D-day 계산 결과 출력  
4. To-do 체크리스트 자동 표시 (`st.checkbox` 리스트로 구현)  
5. 지역 선택 시, 해당 지역 이삿짐센터 리스트(`st.dataframe`) 표시  
6. 필요 시 챗봇 탭에서 “전입신고 언제 해요?” 같은 질문 가능  

---

## 6. 화면 구성 요약 (v1.0 기준)
| 화면 | 주요 요소 |
|------|------------|
| **홈 화면** | D-day 계산기, 체크리스트, 지역 선택 필터 |

---

## 7. 기술 구조 (Streamlit 단일 버전)
- **Framework:** Streamlit (Python 3.10+)  
- **Data:** JSON/CSV 기반 (checklist.json, movers.csv)  
- **Modules:**
  - `datetime` (D-day 계산)
  - `pandas` (데이터 처리)
  - `st.session_state` (상태 관리)
- **배포:**  
  - [Streamlit Community Cloud](https://share.streamlit.io)  
  - 또는 [Hugging Face Spaces](https://huggingface.co/spaces)

## 8. 성공 지표 (Prototype 검증 목표)
- 사용자 5~10명 테스트 후:
